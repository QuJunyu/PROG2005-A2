Help Center

Find answers to common questions and troubleshooting guidance for the Inventory Management System. If you encounter issues not listed here, please refer to the assessment documentation or contact your tutor.

1. Basic System Usage

Q: What devices and browsers does the system support?

A: The system is a fully responsive web application supporting all modern browsers (Chrome 100+, Firefox 98+, Safari 15+, Edge 100+) on both desktop (Windows/Mac/Linux) and mobile (iOS/Android) devices. It is optimized for mobile screens (≤768px) with touch-friendly controls.

Q: Is my inventory data saved permanently?

A: No. The system uses session-persistent storage - all data is stored in your browser's memory only and will be deleted when you close the browser window/tab. No data is saved to your device or any server. For permanent storage, please record inventory data manually.

Q: Can multiple users access the inventory data at the same time?

A: No. The system is a client-side only application with no server-side data sharing. Each browser session has its own independent inventory data.

2. Functional Operations Guide

Q: How do I add a new inventory item?

A: Navigate to the Inventory Operations page, fill in all required fields (marked with *) with valid values, select the correct category and stock status, and click the Submit button. Ensure the Item ID is unique (no duplicates).

Q: How do I edit an existing inventory item?

A: On the Inventory Operations page, enter the Item ID of the item you want to edit (the ID must exist), modify the desired fields (e.g., quantity, price), and click Submit. The system updates the item by its Item Name (case-insensitive matching).

Q: How do I delete an inventory item?

A: On the Inventory Operations page, find the item card you want to delete and click the Delete Item button. A confirmation prompt will appear - click OK to confirm deletion (this action cannot be undone).

Q: How do I search and filter inventory items?

A: Navigate to the Search & Filter page. Type a keyword in the Search by Item Name field for fuzzy name search (case-insensitive). Use the Filter by Category dropdown to filter items by exact category (select All for no filter). Results update in real-time as you type/select.

Q: How do I mark an item as a "Popular Item"?

A: When adding/editing an item on the Inventory Operations page, check the Popular Item checkbox. Popular items are marked with a red badge in all lists and displayed separately on the Search page.

3. Common Errors & Troubleshooting

Q: Why can't I submit the inventory form?

A: The submit button is disabled if the form is invalid. Check for red error messages: all * marked fields are required, quantity/price must be non-negative numbers, and stock status must match the quantity rules (0=Out of Stock, 1-5=Low Stock, >5=In Stock).

Q: Why does the stock status show an error?

A: The system enforces automatic stock status validation. If your selected status does not match the quantity value, the form will reject the submission. Adjust the quantity or select the correct status as prompted.

Q: Why is my new item not showing up?

A: Ensure you clicked the Submit button and saw a green success message. If the item still does not appear, refresh the page or check if you used a duplicate Item ID (duplicate IDs will overwrite existing items).

Q: The search function is not returning results?

A: The search is case-insensitive and matches partial item names. Clear the search keyword and set the category to All to view all items. Check for spelling errors in your search keyword.

Q: The page layout is broken on mobile?

A: The system uses responsive design. Clear your browser cache and ensure you are using a modern browser. The layout automatically adapts to screen sizes down to 320px (mobile phones).

4. System Rules & Constraints

  • Item ID must be unique (no duplicate values allowed).
  • Item Name is used for edit/delete operations (case-insensitive).
  • Quantity and price cannot be negative numbers.
  • Stock status is automatically validated based on quantity.
  • All data is reset when the browser is closed (no persistent storage).
  • Delete actions are permanent and cannot be reversed.
